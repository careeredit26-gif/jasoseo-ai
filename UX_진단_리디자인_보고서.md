# 리뷰 페이지 UX 진단 및 재설계 보고서

---

## 1. 현재 문제점 진단

### 1-1. 랜딩 → 리뷰 페이지 전환의 단절

랜딩 페이지는 "자소서, 붙여넣으면 다듬어져서 나옵니다"라는 강력한 약속을 하고,
기업 맞춤 반영 / 문장별 진단 / 반복 수정이라는 3대 가치를 전달한다.

그런데 로그인 후 리뷰 페이지에 도착하면:
- **약속했던 가치가 어디에도 보이지 않는다.** 빈 폼 3개만 덩그러니 놓여 있다.
- **다크 테마 → 다크 테마** 전환이라 "새로운 화면에 왔다"는 감각이 없다.
- **"자소서 첨삭"이라는 제목만 있고**, 이 화면에서 무엇이 일어나는지 설명이 없다.
- 사용자는 "내가 뭘 입력해야 하고, 결과로 뭘 받는 건지" 1초 안에 파악하지 못한다.

**핵심 문제: 랜딩에서 만든 기대가 리뷰 페이지에서 충족되지 않는다.**

### 1-2. 입력 폼의 문제

| 문제 | 설명 |
|------|------|
| **단계감 부재** | 기업명 / 질문 / 본문이 한 화면에 나열되어 있어 "폼을 채우는 노동"으로 느껴진다 |
| **맥락 없는 필드** | "자소서 질문"이라는 레이블만으로는 뭘 넣어야 하는지 불분명하다 |
| **팁/가이드 없음** | 더 좋은 첨삭 결과를 위해 어떻게 입력해야 하는지 안내가 전혀 없다 |
| **결과 미리보기 없음** | "첨삭 시작하기"를 누르면 무엇이 나오는지 사전 기대를 형성하지 못한다 |
| **글자 수 경고만 있음** | "문항 1개 소진 예상"이라는 표현이 사용자에게 불안감을 준다 |

### 1-3. 결과 화면의 문제

| 문제 | 설명 |
|------|------|
| **빈 상태가 허전함** | 연필 이모지 + "자소서를 입력하고 첨삭을 시작하세요"만 있어 텅 빈 느낌 |
| **정보 우선순위 없음** | 기업 분석 → 문장 진단 → 구조 분석 → 수정본이 동일한 무게로 나열된다 |
| **총평 부재** | 한 줄로 "이 자소서의 핵심 문제는 ~입니다"라는 요약이 없다 |
| **수정 전/후 비교 없음** | 원본과 수정본을 나란히 비교할 수 없다 |
| **추가 수정 UX가 빈약** | 단순 textarea만 있고, 어떤 수정을 요청할 수 있는지 예시가 없다 |

### 1-4. 문구 문제

| 위치 | 수정 전 | 문제 | 수정 후 |
|------|---------|------|---------|
| 페이지 제목 | "자소서 첨삭" | 너무 단순. 기능 설명 없음 | "AI 자소서 첨삭" + 서브: "기업 맞춤 진단부터 수정안까지, 한 번에" |
| 기업 필드 라벨 | "지원 기업" | 왜 입력하는지 이유 없음 | "지원 기업" + 힌트: "기업명을 입력하면 인재상이 자동 반영됩니다" |
| 질문 필드 라벨 | "자소서 질문" | 무엇을 넣는지 불분명 | "자소서 문항" + 힌트: "기업에서 요구한 질문을 그대로 붙여넣으세요" |
| 본문 필드 라벨 | "자소서 내용" | 동기 부여 없음 | "자소서 초안" + 힌트: "완성되지 않아도 괜찮습니다. AI가 다듬어 드립니다" |
| 글자수 표시 | "0자 (문항 1개 소진 예상)" | "소진"이 부정적 | "0 / 800자" + 800자 초과 시 "문항 2개분에 해당합니다" |
| 빈 결과 제목 | "자소서를 입력하고 첨삭을 시작하세요" | 너무 지시적 | "첨삭 결과가 여기에 표시됩니다" |
| 빈 결과 설명 | "기업명과 자소서를 입력하면, AI가 문장별 진단과 구체적 수정안을 제시합니다." | 구조화 안 됨 | 4개 카드로 분리: 기업 적합도 / 문장별 진단 / 구조 분석 / 수정본 |
| 수정 요청 제목 | "추가 수정 요청" | "추가"가 부담감 | "계속 다듬기" |
| 수정 요청 설명 | "수정하고 싶은 부분을 자유롭게 요청하세요" | 예시 부족 | 칩 형태 예시 제공 + "또는 직접 입력" |
| 네비게이션 크레딧 | "문항 5개 · 수정 20회 남음" | "남음"이 소진 압박 | "문항 5개 · 수정 20회 사용 가능" |

## 2. UX 재설계 방향

### 2-1. 테마 전환
- 다크 → **라이트 테마**로 변경
- 이유: 실제 문서 작업 도구(Notion, Google Docs)의 톤에 맞춤
- 랜딩(다크, 마케팅) → 리뷰(라이트, 작업 도구) 전환으로 "이제 실제로 쓰는 단계"임을 체감

### 2-2. 정보 구조 재설계

```
┌─────────────────────────────────────────────────┐
│ [상단 바]                                        │
│  로고 · 문항 ○개 / 수정 ○회 사용 가능 · 로그아웃  │
├─────────────────────────────────────────────────┤
│ [상단 안내]                                      │
│  AI 자소서 첨삭                                   │
│  기업 맞춤 진단부터 수정안까지, 한 번에             │
│  ───── 스텝 인디케이터 ─────                     │
│  ① 기업 선택  ② 문항 입력  ③ 초안 작성  ④ 결과    │
├──────────────┬──────────────────────────────────┤
│ [입력 패널]   │ [결과 패널]                       │
│              │                                   │
│ 기업명 입력   │ (빈 상태 → 4개 미리보기 카드)      │
│ ↓            │  · 기업 적합도 분석                │
│ 문항 입력     │  · 문장별 진단                    │
│ ↓            │  · 구조 점수                      │
│ 초안 붙여넣기 │  · AI 수정본                      │
│ ↓            │                                   │
│ [첨삭 시작]   │ (결과 시) 총평 → 점수 → 진단 →    │
│              │   Before/After → 수정본 → 재수정   │
│ [첨삭 기록]   │                                   │
└──────────────┴──────────────────────────────────┘
```

### 2-3. 핵심 개선 사항

1. **스텝 인디케이터**: 입력 과정에 단계감 부여
2. **빈 상태 → 미리보기 카드 4개**: 결과로 뭘 받는지 사전 기대 형성
3. **입력 필드마다 힌트 텍스트**: 왜 이걸 입력해야 하는지 설명
4. **팁 박스**: "더 좋은 결과를 위한 팁" 접이식 제공
5. **결과 화면 정보 계층**: 총평 → 점수 → Top 3 문제 → 비교 → 수정본 → 재수정
6. **수정 요청 칩**: 예시 칩 5~6개 제공, 탭하면 자동 입력
7. **Before/After 나란히 비교**: 원본과 수정본을 좌우 또는 상하로 비교

## 3. 결과 화면 렌더링 순서 (JS 변경)

### 현재 순서:
1. 기업 분석 → 2. 문장 진단 → 3. 구조 점수 → 4. 수정본 → 5. 요약 → 6. 수정 요청

### 개선 순서:
1. **총평 (한 줄 요약)** — 가장 먼저 "이 자소서의 핵심 문제"를 보여준다
2. **핵심 점수 4개** — 흐름/논리/구체성/기업 적합도를 스코어카드로
3. **기업 분석** — 인재상 반영 전략
4. **Top 3 핵심 문제** — 가장 심각한 문제 3개만 우선 표시
5. **Before/After 비교** — 원본과 수정본을 나란히
6. **전체 수정본** — 복사 가능한 최종본
7. **계속 다듬기** — 칩 예시 + 자유 입력

이 순서는 "결론 먼저 → 근거 → 실행"의 피라미드 구조를 따른다.
사용자는 스크롤하면서 점점 더 깊은 분석에 접근하되,
첫 화면에서 "이 자소서가 어떤 상태인지"를 1초 안에 파악할 수 있다.
